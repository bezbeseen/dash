Card Style Lab — self-contained playground. All CSS is inlined so it works via file://. When you settle on a look, copy rules into app/globals.css (search for .card, .job-card-*, .btn).

Other labs: Ticket Detail • Board & Columns • Buttons & UI Kit

What each class is for
.card White panel: border, padding, grid layout, shadow. Maps to outer wrapper in JobCard.
.card-main-link Whole ticket preview is one link to the job detail page. Hover/focus ring; grid stacks title, lines, badge, hint.
.meta Muted helper text elsewhere in the app. Job card detail lines use .job-card-estimate (etc.) in app/globals.css, not .meta.
.job-card-title Customer / primary heading (bold inside).
.job-card-subtitle Product or job line (e.g. “500 Business Cards”). Omitted when empty.
.job-card-estimate Estimate amount line.
.job-card-invoice Paid vs invoice total line.
.job-card-quickbooks QuickBooks date + Dash created timestamp.
.job-card-updated Last updated line.
.job-card-extra Optional extra line from the board; also .card-extra-meta.
.job-card-status + .badge Status pill (e.g. QUOTED). Keep both classes: .badge supplies the look.
.job-card-open-hint “Open ticket →”; pairs with .card-open-hint (focus/hover behavior).
.job-card-action … Each <form> has .job-card-action plus .job-card-action-start|ready|delivered|done|lost for debugging or CSS hooks.
.actions Flex row of forms/buttons with gap and wrap.
.actions-card Same row, but z-index so buttons stay clickable above the big link (stacking context). From JobCardActions.
.btn · .btn-done · .btn-lost Custom buttons (not Bootstrap variants). Done = green tint, Lost = red tint.
